##  Теоретические основы

### Зачем нужна диагностика?
Специализированное ПО для диагностики решает несколько ключевых задач:
- **Инвентаризация** — получение полной информации о конфигурации (память, диски, периферия)
- **Контроль работоспособности** — проверка корректности работы устройств
- **Предиктивная аналитика** — выявление ранних признаков износа (например, деградация поверхности HDD)
- **Защита данных** — предотвращение потерь информации благодаря своевременному обнаружению проблем

### Объекты тестирования
Диагностика может охватывать как систему целиком, так и отдельные модули:
- Материнская плата и чипсет
- Оперативная память (различные типы и конфигурации)
- Видеоподсистема (2D/3D ускорение)
- Накопители (HDD, SSD, оптические приводы)
- Устройства ввода (клавиатура, мышь)
- Интерфейсы (USB, COM, LPT, сетевые адаптеры)

## 🛠 Используемые инструменты

### 1. PC Wizard 2014
Утилита для детального анализа конфигурации и тестирования производительности.

**Возможности:**
- Сбор информации об аппаратном обеспечении и ОС
- Бенчмарк процессора, памяти, дисков
- Тестирование графической подсистемы
- Поддержка 32- и 64-битных систем

### 2. Средство диагностики DirectX (dxdiag)
Встроенная утилита Windows для проверки компонентов мультимедиа.

**Функционал:**
- Информация о драйверах DirectX
- Проверка видео- и аудиоадаптеров
- Диагностика устройств ввода
- Отчёт о системных проблемах и конфликтах

## 📋 Ход работы

### Часть 1: Тестирование в PC Wizard

1.  Развёрнута виртуальная машина с Windows 7 (для совместимости с PC Wizard 2014)
2.  Установлен дистрибутив `pc-wizard_2014.2.14-setup.exe`
3.  Запущена программа, активирован раздел **ТЕСТ**
4.  Поочерёдно выполнены тесты компонентов (за исключением GLOBAL PERFORMANCE)
5.  Результаты зафиксированы в протоколе

### Часть 2: Диагностика DirectX

1.  Вызвано окно «Выполнить» комбинацией `Win + R`
2.  Введена команда `dxdiag`
3.  Последовательно проверены все вкладки: **Система → Экран → Звук → Ввод**
4.  В поле «Примечания» каждой вкладки подтверждено: **«Неполадок не найдено»**
5.  Собрана информация о конфигурации системы

## ❓ Контрольные вопросы


<summary><b>1. Основные причины проблем с совместимостью ПО</b></summary>

- Устаревшее аппаратное обеспечение, не поддерживающее новые инструкции или стандарты
- Конфликты между версиями библиотек и драйверов
- Отсутствие актуальных обновлений ОС и приложений
- Различия в архитектурах (32-bit vs 64-bit)
- Несоответствие минимальным системным требованиям



<summary><b>2. Методы снижения проблем совместимости</b></summary>

- Своевременная установка обновлений Windows и драйверов устройств
- Использование режимов совместимости для старых программ
- Регулярный апгрейд аппаратных компонентов
- Проверка системных требований перед установкой ПО
- Применение виртуальных машин для запуска несовместимых приложений
