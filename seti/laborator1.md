### 4.1. История и стандарты Ethernet

#### История развития:
```
📅 1975 г. — Разработка стандарта Ethernet в Xerox PARC
📅 1980 г. — Публикация "Blue Book Standard" для Ethernet I (DIX)
📅 1985 г. — Принятие стандарта IEEE 802.3
📅 1995 г. — Fast Ethernet (IEEE 802.3u) — 100 Мбит/с
📅 1998 г. — Gigabit Ethernet (IEEE 802.3z) — 1 Гбит/с
📅 2002 г. — 10 Gigabit Ethernet (IEEE 802.3ae) — 10 Гбит/с
```

#### Модификации стандарта IEEE 802.3:

| Стандарт | Среда передачи | Макс. длина сегмента | Скорость | Статус |
|----------|---------------|---------------------|----------|--------|
| **10Base-5** | Коаксиал "толстый" (50 Ом) | 500 м | 10 Мбит/с | ❌ Устарел |
| **10Base-2** | Коаксиал "тонкий" (50 Ом) | 185 м | 10 Мбит/с | ❌ Устарел |
| **10Base-T** | UTP Cat.3+ (витая пара) | 100 м | 10 Мбит/с | ✅ Актуален |
| **10Base-F** | Оптоволокно | 1000-2000 м | 10 Мбит/с | ✅ Специализированный |
| **100Base-TX** | UTP Cat.5 (2 пары) | 100 м | 100 Мбит/с | ✅ Основной |
| **1000Base-T** | UTP Cat.5e/6 (4 пары) | 100 м | 1 Гбит/с | ✅ Основной |

#### Метод доступа к среде: CSMA/CD
```
CSMA/CD = Carrier Sense Multiple Access with Collision Detection
(Множественный доступ с контролем несущей и обнаружением коллизий)

🔄 Алгоритм работы:
1. Прослушивание канала (Carrier Sense)
2. Если канал свободен → передача данных
3. Если канал занят → ожидание случайного интервала
4. Обнаружение коллизии → немедленное прекращение передачи
5. Отправка "jam signal" для уведомления всех узлов
6. Повторная попытка после случайной задержки (backoff)
```

### 4.2. Структурированные кабельные системы (СКС)

#### Определение:
> **СКС** — универсальная телекоммуникационная инфраструктура здания, обеспечивающая передачу сигналов всех типов: речевых, информационных, видео.

#### Преимущества СКС перед устаревшими системами:

```
✅ Интеграция: передача разнотипных сигналов по единой инфраструктуре
✅ Долговечность: работа с несколькими поколениями оборудования
✅ Универсальность: подключение любого совместимого оборудования
✅ Масштабируемость: скорость от 100 Кбит/с до 10 Гбит/с
✅ Экономичность: снижение эксплуатационных расходов
✅ Гибкость: свобода перемещения пользователей без изменения настроек
✅ Прозрачность: маркировка и документация всех интерфейсов
```

#### Подсистемы СКС (по ISO/IEC 11801):

```
🏢 СКС здания
│
├── 📡 Магистральная подсистема комплекса зданий
│   └── Соединение кабельных систем разных зданий
│
├── 🏗️ Магистральная подсистема здания
│   ├── Информационная подсистема (оптоволокно/витая пара)
│   └── Речевая подсистема (многопарные кабели)
│
└── 📐 Горизонтальная подсистема (90% всех кабелей)
    ├── Распределительные панели
    ├── Горизонтальные кабели (до 90 м)
    ├── Точки консолидации
    └── Телекоммуникационные розетки
```

#### Категории кабелей для горизонтальной подсистемы:

| Категория | Полоса частот | Макс. скорость | Применение |
|-----------|--------------|----------------|------------|
| **Cat.3** | 16 МГц | 10 Мбит/с | Телефония, 10Base-T |
| **Cat.5** | 100 МГц | 100 Мбит/с | 100Base-TX |
| **Cat.5e** | 125 МГц | 1 Гбит/с | ✅ Основной стандарт |
| **Cat.6** | 250 МГц | 10 Гбит/с* | ЦОД, новые здания |
| **Cat.6a** | 500 МГц | 10 Гбит/с | Высокоскоростные сети |
| **Cat.7** | 600-700 МГц | 10 Гбит/с | Экранированные решения |

> *Для 10 Гбит/с по Cat.6 ограничение расстояния — 55 м

### 4.3. Типы сетевых кабелей

#### 🔹 Коаксиальный кабель

```
📐 Конструкция:
┌─────────────────────────────┐
│  [A] Внешняя оболочка       │ ← ПВХ/полиэтилен, защита
│  [B] Экран (оплётка/фольга) │ ← Медь/алюминий, защита от помех
│  [C] Диэлектрическая изоляция│ ← Полиэтилен, фиксация жилы
│  [D] Центральный проводник  │ ← Медь/омеднённая сталь
└─────────────────────────────┘

⚡ Волновое сопротивление:
• 50 Ом — радиосвязь, компьютерные сети (10Base2, 10Base5)
• 75 Ом — телевидение, видеосигналы (RG-6)
• 93 Ом — импульсная техника (RG-62, ARCNet)

🔄 Принцип стоячей волны:
• Совпадение центров проводников + определённое соотношение диаметров
• → Снижение потерь на излучение до ~0%
• → Экран защищает от внешних электромагнитных помех
```

#### 🔹 Витая пара (Twisted Pair)

```
📐 Конструкция:
• 4 пары изолированных медных проводников (0.4-0.6 мм)
• Проводники скручены с разным шагом для снижения перекрёстных наводок
• Внешняя оболочка из ПВХ/полипропилена/вспененного полиэтилена

🔒 Типы экранирования:
┌─────────┬─────────────────────────────────┐
│ Аббр.   │ Описание                          │
├─────────┼─────────────────────────────────┤
│ UTP     │ Unshielded — без экранирования   │
│ FTP     │ Foiled — общий экран из фольги   │
│ STP     │ Shielded — экран для каждой пары │
│ S/FTP   │ Экран + фольга на парах          │
│ S/STP   │ Двойное экранирование            │
└─────────┴─────────────────────────────────┘

📊 Одножильный vs Многожильный кабель:
┌────────────────┬─────────────────────────┐
│ Одножильный    │ Многожильный            │
├────────────────┼─────────────────────────┤
│ • Жёсткий      │ • Гибкий                │
│ • Для стационарной прокладки │ • Для патч-кордов │
│ • Лучший контакт в розетках │ • Не для "врезания" │
│ • Меньшее затухание │ • Большее затухание │
└────────────────┴─────────────────────────┘
```

#### 🔹 Оптоволоконный кабель

```
💡 Принцип работы:
• Полное внутреннее отражение света в стеклянной/пластиковой нити
• Передача данных модулированным световым сигналом

📐 Типы оптоволокна:
┌─────────────────┬────────────────────────┐
│ Одномодовое     │ Многомодовое           │
├─────────────────┼────────────────────────┤
│ • Диаметр жилы: 8-10 мкм │ • Диаметр: 50/62.5 мкм │
│ • Один луч света │ • Множество лучей (мод) │
│ • Дальность: до 100 км │ • Дальность: до 500 м │
│ • Дороже       │ • Дешевле              │
│ • Для магистралей │ • Для локальных сетей │
└─────────────────┴────────────────────────┘

✅ Преимущества перед медными кабелями:
• Широкая полоса пропускания (Тбит/с)
• Минимальное затухание (0.2-0.35 дБ/км)
• Полная невосприимчивость к электромагнитным помехам
• Гальваническая развязка (безопасность)
• Высокая защита от несанкционированного доступа
• Малый вес и объём
• Взрыво- и пожаробезопасность
```

### 4.4. Кабель «Витая пара» — детальный разбор

#### Цветовая маркировка проводов (8-жильный кабель):

```
🎨 Стандартная расцветка по парам:

Пара 1 (Бело-синяя):    🔵 Белый/Синий  +  🔵 Синий
Пара 2 (Бело-оранжевая): 🟠 Белый/Оранжевый + 🟠 Оранжевый  
Пара 3 (Бело-зелёная):   🟢 Белый/Зелёный + 🟢 Зелёный
Пара 4 (Бело-коричневая):🟤 Белый/Коричневый + 🟤 Коричневый

📋 Для 10/100 Мбит/с используются только 2 пары:
• Пара 2 (оранжевая): контакты 1-2 → передача (TX+/TX-)
• Пара 3 (зелёная): контакты 3-6 → приём (RX+/RX-)

📋 Для 1 Гбит/с используются все 4 пары:
• Все 8 контактов задействованы для двунаправленной передачи
```

#### Технические характеристики кабеля UTP Cat.5e:

```
📊 Параметры:
• Активное сопротивление: ≤ 9.4 Ом на 100 м
• Волновое сопротивление: 100 ± 25 Ом (на частоте 100-120 МГц)
• Затухание сигнала: 0.8-22 дБ (частоты 64 кГц — 100 МГц)
• Максимальная длина сегмента: 100 м (включая патч-корды)
• Минимальный радиус изгиба: 8× внешний диаметр кабеля

⚠️ Важные замечания:
• Не снимать изоляцию с отдельных жил при обжиме!
• Избегать "скруток" и резких изгибов кабеля
• Для экранированных кабелей — обеспечивать целостность экрана
• Дренажный провод должен быть соединён с экраном разъёма
```

### 4.5. Разъёмы и коннекторы

#### 🔌 Разъём 8P8C (ошибочно называемый "RJ-45")

```
⚠️ Важное уточнение:
• Правильное название: 8P8C (8 Position 8 Contact)
• "RJ-45" — это стандарт для телефонных разъёмов 8P4C с ключом
• Термин "RJ-45" прижился в ИТ-среде, но технически неверен

📐 Конструкция коннектора 8P8C:
┌─────────────────────────────┐
│  [1] Корпус из прозрачного пластика │
│  [2] 8 позолоченных контактов (ножей)│
│  [3] Фиксатор кабеля (защёлка)      │
│  [4] Направляющие для проводов      │
│  [5] Стопор для фиксации оболочки   │
└─────────────────────────────┘

🔧 Типы разъёмов (по количеству контактов):
• 8P8C — 8 контактов, для ЛВС (основной)
• 6P6C — 6 контактов, RJ-25, телефония
• 6P4C — 4 контакта, RJ-14, телефонные трубки
• 4P4C — 4 контакта, RJ-9/10/11, гарнитуры
```

#### 📡 Порты MDI и MDIX

```
🔄 Назначение портов:

┌─────────────┬─────────────────────────────┐
│ Тип порта   │ Где применяется             │
├─────────────┼─────────────────────────────┤
│ MDI         │ Сетевые адаптеры ПК, серверы│
│ (Medium Dependent Interface) │ (конечные устройства) │
├─────────────┼─────────────────────────────┤
│ MDI-X       │ Коммутаторы, концентраторы  │
│ (MDI Crossover) │ (активное оборудование) │
└─────────────┴─────────────────────────────┘

🔗 Типы кабелей для соединения:

1️⃣ ПРЯМОЙ КАБЕЛЬ (Straight):
   • Соединение: MDI ↔ MDIX
   • Применение: ПК → Коммутатор, ПК → Роутер
   • Схема: одинаковая на обоих концах (568A или 568B)

2️⃣ ПЕРЕКРЁСТНЫЙ КАБЕЛЬ (Crossover):
   • Соединение: MDI ↔ MDI или MDIX ↔ MDIX
   • Применение: ПК ↔ ПК, Коммутатор ↔ Коммутатор
   • Схема: 568A на одном конце, 568B на другом

💡 Современные устройства поддерживают Auto-MDI/MDIX:
• Автоматическое определение типа кабеля
• Исключает ошибку выбора прямого/перекрёстного кабеля
```

### 4.6. Схемы обжима кабеля

#### 📋 Схема обжима по стандарту **EIA/TIA-568B** (НАИБОЛЕЕ РАСПРОСТРАНЕННАЯ):

```
🔢 Расположение контактов (вид на коннектор контактами ВВЕРХ, защёлкой ВНИЗ):

┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │ ← Номер контакта
├───┼───┼───┼───┼───┼───┼───┼───┤
│Б-О│ О │Б-З│ С │Б-С│ З │Б-К│ К │ ← Цвет провода
└───┴───┴───┴───┴───┴───┴───┴───┘

🎨 Условные обозначения:
• Б-О = Белый с оранжевой полосой
• О  = Оранжевый (сплошной)
• Б-З = Белый с зелёной полосой
• С  = Синий (сплошной)
• Б-С = Белый с синей полосой
• З  = Зелёный (сплошной)
• Б-К = Белый с коричневой полосой
• К  = Коричневый (сплошной)
```

#### 📋 Схема обжима по стандарту **EIA/TIA-568A** (альтернативная):

```
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ 7 │ 8 │
├───┼───┼───┼───┼───┼───┼───┼───┤
│Б-З│ З │Б-О│ С │Б-С│ О │Б-К│ К │
└───┴───┴───┴───┴───┴───┴───┴───┘

🔄 Разница между 568A и 568B:
• Поменяны местами оранжевая и зелёная пары
• Для прямого кабеля: использовать ОДИНАКОВУЮ схему на обоих концах
• Для перекрёстного: 568A на одном конце, 568B на другом
```

#### 🔀 Перекрёстный кабель (Crossover) для соединения ПК↔ПК:

```
📐 Схема для 10/100 Мбит/с (4 провода):

КОНЕЦ 1 (568B)          КОНЕЦ 2 (568A)
┌────────────┐         ┌────────────┐
│1: Б-О ─────┼─────────┼── 3: Б-З    │
│2: О  ──────┼─────────┼── 6: З      │
│3: Б-З ─────┼─────────┼── 1: Б-О    │
│6: З  ──────┼─────────┼── 2: О      │
└────────────┘         └────────────┘

📐 Схема для 1 Гбит/с (8 проводов — полная перекрёстка):
• Все 4 пары меняются местами согласно 568A↔568B
```

#### ⚡ "Ленивая" обжимка на 2 пары (только для 100 Мбит/с):

```
✅ Когда можно использовать:
• Для устройств, не требующих скорости >100 Мбит/с
• Для временных соединений
• При экономии коннекторов

⚠️ Недостатки:
• Невозможность работы на 1 Гбит/с
• При апгрейде сети потребуется переобжим всех кабелей
• Меньшая надёжность (не задействованы все контакты)

📋 Используемые контакты:
• Контакты 1,2,3,6 (оранжевая и зелёная пары)
• Контакты 4,5,7,8 можно оставить свободными или использовать для второго канала
```

### 4.7. Сетевые адаптеры и MAC-адреса

#### 🔧 Конструкция сетевой карты (NIC):

```
📐 Основные компоненты:
┌─────────────────────────────┐
│ [1] Разъём RJ-45           │ ← Подключение кабеля
│ [2] LED-индикаторы         │ ← Link/Activity статус
│ [3] Интерфейс PCI/PCIe     │ ← Подключение к материнской плате
│ [4] Слот BootROM           │ ← Загрузка по сети (PXE)
│ [5] Контроллер сетевого чипа│ ← Обработка трафика
│ [6] Коннектор Wake-on-LAN  │ ← Удалённое включение ПК
└─────────────────────────────┘
```

#### 🆔 MAC-адрес (Media Access Control):

```
📋 Характеристики:
• Длина: 48 бит (6 байт)
• Формат записи: 12:34:56:78:90:AB (шестнадцатеричный)
• Назначается производителем при изготовлении
• Должен быть уникальным в глобальном масштабе

🏭 Структура адреса:
┌─────────────────────────────┐
│ [24 бита] [24 бита]         │
│   ↓          ↓              │
│  OUI      NIC-specific      │
│ (произ-  (серийный номер   │
│ водитель)  устройства)      │
└─────────────────────────────┘

📊 Примеры OUI (первые 3 байта):
• 00:00:0C — Cisco
• 00:00:0E — Fujitsu
• 00:80:C8 — D-Link
• 00:A0:24 — 3Com
• 00:C0:49 — US Robotics

🔄 Работа адаптера в сети:
1. Прослушивание всего трафика в сегменте
2. Сравнение поля Destination MAC с собственным адресом
3. Обработка кадров, адресованных себе (или широковещательных)
4. Игнорирование остальных кадров

⚙️ Программное изменение MAC:
• Возможно через драйвер/ОС для тестирования
• Используется для: анонимизации, обхода фильтрации, тестирования
• ⚠️ Может нарушать политики безопасности сети
```

#### ⚙️ Конфигурирование сетевой платы:

```
🔧 Параметры, требующие настройки (для старых карт):
• Адрес порта ввода-вывода (I/O Port Address)
• Номер прерывания (IRQ)
• Адрес памяти (Memory Address) — для DMA

✅ Современные карты (Plug and Play):
• Автоматическое определение ресурсов
• Настройка через BIOS/ОС
• Драйверы для Windows/Linux устанавливаются автоматически

🔍 Проверка конфигурации в Windows:
• Диспетчер устройств → Сетевые адаптеры
• Свойства адаптера → вкладка "Ресурсы"
• Команда: `ipconfig /all` в командной строке
```

### 5.2. Пошаговая инструкция по обжиму кабеля

#### 🛠️ Этап 1: Подготовка кабеля

```
1️⃣ Отмерить необходимую длину кабеля
   • Оставить запас 1-1.5 м с каждой стороны для монтажа
   • Отрезать кабель кусачками под прямым углом

2️⃣ Снять внешнюю изоляцию:
   • Использовать стриппер или нож
   • Длина зачистки: 12-15 мм
   • ⚠️ НЕ повредить изоляцию внутренних жил!
   
   ┌─────────────────┐
   │  [====]        │ ← Внешняя оболочка
   │  |    |        │ ← Зачищенный участок (12-15 мм)
   │  |🔵🟠🟢🟤|   │ ← Пары проводов
   │  [====]        │ ← Оставшаяся оболочка
   └─────────────────┘

3️⃣ Раскрутить пары и выпрямить провода:
   • Аккуратно расплести каждую пару
   • Выпрямить провода пальцами
   • Не растягивать и не перегибать
```

#### 🛠️ Этап 2: Расположение проводов по схеме

```
4️⃣ Упорядочить провода по выбранной схеме (568B рекомендуется):

   Порядок слева направо (вид на провода со стороны среза):
   
   1️⃣ 🟠⬜ Белый/Оранжевый
   2️⃣ 🟠  Оранжевый
   3️⃣ 🟢⬜ Белый/Зелёный
   4️⃣ 🔵  Синий
   5️⃣ 🔵⬜ Белый/Синий
   6️⃣ 🟢  Зелёный
   7️⃣ 🟤⬜ Белый/Коричневый
   8️⃣ 🟤  Коричневый

5️⃣ Выровнять провода:
   • Сжать провода вместе плоской поверхностью
   • Проверить порядок цветов
   • Обрезать провода на одном уровне (оставить ~12-14 мм)
   
   💡 Совет: использовать лезвие обжимного инструмента для точного среза
```

#### 🛠️ Этап 3: Установка в коннектор и обжим

```
6️⃣ Вставить провода в коннектор 8P8C:
   • Держать коннектор защёлкой ВНИЗ, контактами К СЕБЕ
   • Вводить провода до упора в каналы коннектора
   • Контролировать, чтобы:
     ✓ Каждый провод попал в свой канал
     ✓ Провода дошли до передней стенки коннектора
     ✓ Внешняя оболочка зашла под фиксатор (на 3-5 мм)

   ✅ Правильное положение:
   ┌─────────────────────────┐
   │  🔩🔩🔩🔩🔩🔩🔩🔩  │ ← Контакты (вид сверху)
   │  🟠⬜🟠🟢⬜🔵🔵⬜🟢🟤⬜🟤 │ ← Провода
   │  [================]    │ ← Оболочка под фиксатором
   │  └─────🔒─────┘        │ ← Защёлка
   └─────────────────────────┘

7️⃣ Обжать коннектор:
   • Вставить коннектор в гнездо обжимного инструмента
   • Плавно, но с усилием сжать рукоятки до щелчка
   • Контролировать, что:
     ✓ Все 8 контактов утоплены и пробили изоляцию проводов
     ✓ Фиксатор кабеля зажал внешнюю оболочку
     ✓ Коннектор не деформирован

   ⚠️ Если нет обжимного инструмента (экстренный случай):
   • Использовать тонкую отвёртку
   • Поочерёдно утопить каждый контакт (1→8)
   • Затем зафиксировать кабель фиксатором
   • ⚠️ Надёжность ниже, использовать только для тестов!
```

#### 🛠️ Этап 4: Повторить для второго конца

```
8️⃣ Для ПРЯМОГО кабеля:
   • Повторить шаги 1-7 с использованием ТОЙ ЖЕ схемы (568B)

9️⃣ Для ПЕРЕКРЁСТНОГО кабеля:
   • На втором конце использовать схему 568A
   • Или вручную переставить пары: 1↔3, 2↔6

🔟 Маркировка готового кабеля:
   • Подписать концы (если требуется)
   • Зафиксировать дату изготовления
   • Указать тип кабеля (прямой/перекрёстный)
```

### 5.3. Проверка качества соединения

#### 🔍 Визуальный контроль:

```
✅ Проверить:
• Все провода дошли до конца коннектора
• Цвета соответствуют схеме на обоих концах

• Контакты равномерно утоплены, нет погнутых

```

#### 🔌 Тестирование кабельным тестером:

```
📱 Использование кабельного тестера:

1. Подключить один конец кабеля к основному блоку (Master)
2. Подключить второй конец к удалённому блоку (Remote)
3. Включить тестер

📊 Интерпретация результатов:

✅ УСПЕШНЫЙ ТЕСТ:
• Светодиоды 1→8 загораются последовательно на обоих блоках
• Индикаторы синхронизированы
• Нет пропущенных или мигающих светодиодов

❌ ОШИБКИ И ИХ ПРИЧИНЫ:

┌─────────────┬────────────────────────────┐
│ Сигнал      │ Возможная причина          │
├─────────────┼────────────────────────────┤
│ 🔴 Нет свечения на №3 │ Обрыв провода 3    │

└─────────────┴────────────────────────────┘

🔧 Устранение неисправностей:
• Переобжать проблемный конец кабеля
• Проверить целостность жил омметром
• Заменить коннектор при повреждении
```

#### 🧪 Тестирование омметром (если нет тестера):

```
📐 Проверка целостности:
1. Перевести мультиметр в режим прозвонки (Ω)
2. Проверить каждую жилу отдельно:
   • Контакт 1 на конце A ↔ Контакт 1 на конце B
   • ... повторить для контактов 2-8
3. Проверить отсутствие замыканий:
   • Контакт 1 ↔ Контакты 2-8 (должно быть ∞ Ом)
   • Повторить для всех комбинаций

✅ Нормальные показания:
• Сопротивление жилы: < 10 Ом на 100 м
• Изоляция между жилами: > 1 МОм
```

## ❓ Контрольные вопросы

### Блок 1: Кабели и стандарты

```
1️⃣ Какие сетевые кабели использует технология Ethernet?
   ✅ Ответ: Коаксиальный (10Base2, 10Base5), витая пара (10Base-T и новее), 
   оптоволокно (10Base-F). Современный стандарт — витая пара категории 5e/6.

2️⃣ Что такое кабель UTP? В чём его достоинства и недостатки?
   ✅ Ответ: 
   • UTP = Unshielded Twisted Pair — неэкранированная витая пара
   • ✅ Достоинства: дешёвый, гибкий, простой в монтаже, достаточная 
     производительность для большинства задач
   • ❌ Недостатки: чувствителен к электромагнитным помехам, 
     ограниченная длина сегмента (100 м)

3️⃣ Почему при монтаже вилки RJ-45 нет необходимости снимать изоляцию 
   с отдельных жил кабеля?
   ✅ Ответ: Контакты коннектора 8P8C выполнены в виде острых ножей, 
   которые при обжиме прорезают изоляцию провода и обеспечивают 
   надёжный электрический контакт с медной жилой. Снятие изоляции 
   вручную может привести к повреждению жилы и ухудшению контакта.

4️⃣ Что такое «нуль-модемный» кабель и для каких целей он применяется?
   ✅ Ответ: 
   • Нуль-модемный (перекрёстный, crossover) кабель — кабель, в котором 
     на одном конце использована схема 568A, на другом — 568B
   • Применяется для прямого соединения двух однотипных устройств: 
     ПК↔ПК, коммутатор↔коммутатор, без использования промежуточного 
     активного оборудования
```

### Блок 2: Сетевые устройства и адреса

```
5️⃣ Что такое сетевые устройства MDI и MDIX? Для соединения каких устройств 
   необходим перекрёстный кабель?
   ✅ Ответ:
   • MDI (Medium Dependent Interface) — порт конечного устройства 
     (сетевая карта ПК, сервера)
   • MDIX (MDI Crossover) — порт активного оборудования 
     (коммутатор, концентратор) с перекрёстным расположением контактов
   • Перекрёстный кабель нужен при соединении: MDI↔MDI или MDIX↔MDIX
   • Современные устройства с Auto-MDI/MDIX автоматически определяют 
     тип кабеля, делая перекрёстные кабели необязательными

6️⃣ Каким образом однозначно идентифицируются сетевые адаптеры? 
   С какой целью введена возможность изменения MAC-адреса?
   ✅ Ответ:
   • Идентификация: по уникальному 48-битному MAC-адресу, 
     назначаемому производителем (первые 24 бита = OUI производителя)
   • Изменение возможно программно для:
     ✓ Тестирования сетевых функций и безопасности
     ✓ Анонимизации в публичных сетях
     ✓ Обхода фильтрации по MAC-адресам (не рекомендуется)
     ✓ Восстановления доступа при утере оригинального адреса

7️⃣ В чём заключается процесс конфигурирования сетевой платы? 
   Какие параметры при этом настраиваются?
   ✅ Ответ:
   • Процесс: установка карты → загрузка ОС → обнаружение устройства → 
     установка драйверов → настройка параметров сети
   • Настраиваемые параметры:
     ✓ Прерывание (IRQ) — для старых карт
     ✓ Адрес порта ввода-вывода (I/O Address)
     ✓ MAC-адрес (опционально)
     ✓ Скорость и дуплекс (Auto/10/100/1000 Мбит/с, Half/Full)
     ✓ Параметры TCP/IP (IP-адрес, маска, шлюз, DNS) — на уровне ОС
```

### Блок 3: Практические аспекты

```
8️⃣ Каковы основные этапы проверки качества обжатого кабеля?
   ✅ Ответ:
   1. Визуальный осмотр: правильность порядка цветов, фиксация оболочки
   2. Тестирование кабельным тестером: последовательное свечение 1→8
   3. Проверка омметром: целостность жил, отсутствие замыканий
   4. Функциональный тест: подключение к сети, проверка связи (ping)

9️⃣ Почему важно оставлять запас кабеля 1-1.5 м при монтаже?
   ✅ Ответ:
   • Возможность повторного обжима при ошибке или повреждении
   • Гибкость при перемещении оборудования
   • Компенсация натяжения и вибраций
   • Удобство укладки и кабель-менеджмента
   • Соответствие требованиям стандартов СКС

🔟 Какие меры предосторожности следует соблюдать при работе с кабелем?
   ✅ Ответ:
   • Не превышать минимальный радиус изгиба (8× диаметр кабеля)
   • Не натягивать кабель с усилием >110 Н (риск повреждения жил)
   • Защищать от механических повреждений, влаги, УФ-излучения
   • Для экранированных кабелей — сохранять целостность экрана
   • Использовать антистатические меры при работе с электроникой
```

## 📎 Приложения

### Приложение А: Шпаргалка по цветам (568B)

```
🎨 Схема 568B — запомнить порядок:

Контакт:  1    2    3    4    5    6    7    8
         ↓    ↓    ↓    ↓    ↓    ↓    ↓    ↓
Цвет:   Б-О   О   Б-З   С   Б-С   З   Б-К   К

🧠 Мнемоническое правило:
"Оранжевые первыми идут, потом зелёные, 
синие в центре, коричневые в конце"

🔄 Для 568A — поменять местами оранжевую и зелёную пары:
Контакт:  1    2    3    4    5    6    7    8
Цвет:   Б-З   З   Б-О   С   Б-С   О   Б-К   К
```

### Приложение Б: Чек-лист перед сдачей работы

```
✅ ФИНАЛЬНАЯ ПРОВЕРКА:

[ ] Кабель обрезан ровно, без заусенцев
[ ] Внешняя изоляция снята на 12-15 мм, жилы не повреждены
[ ] Провода расположены в правильном порядке (проверить дважды!)
[ ] Провода обрезаны на одном уровне (~12-14 мм от среза оболочки)
[ ] Провода вставлены в коннектор до упора
[ ] Внешняя оболочка зашла под фиксатор коннектора
[ ] Все 8 контактов равномерно утоплены при обжиме
[ ] Фиксатор кабеля надёжно зажал оболочку
[ ] Коннектор не имеет трещин или деформаций
[ ] Тестер показывает последовательное свечение 1→8 на обоих концах
[ ] Нет замыканий между контактами (проверка омметром)
[ ] Кабель промаркирован (тип, дата, исполнитель)
[ ] Рабочее место убрано, инструменты сложены

🎯 Готово к сдаче! ✅
```

### Приложение В: Типичные ошибки и их исправление

```
❌ ОШИБКА: Провода не дошли до конца коннектора
   🔧 Решение: Переобжать, контролировать глубину вставки

❌ ОШИБКА: Неправильный порядок цветов
   🔧 Решение: Проверять схему перед обрезкой, использовать шпаргалку

❌ ОШИБКА: Повреждена изоляция внутренних жил
   🔧 Решение: Аккуратнее работать со стриппером, не давить на нож

❌ ОШИБКА: Внешняя оболочка не зафиксирована под зажимом
   🔧 Решение: Оставить больше оболочки при зачистке, контролировать при вставке

❌ ОШИБКА: Контакты не пробили изоляцию проводов
   🔧 Решение: Сильнее сжать обжимной инструмент до характерного щелчка

❌ ОШИБКА: Коннектор треснул при обжиме
   🔧 Решение: Проверить правильность установки в гнездо инструмента, 
   не использовать повреждённые коннекторы
```
